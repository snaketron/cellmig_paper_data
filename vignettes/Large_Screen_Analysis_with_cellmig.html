1 Introduction

In large-scale chemical screens with hundreds of compounds, controlling false positives is critical. cellmig uses Bayesian hierarchical modeling with partial pooling (shrinkage) to automatically regularize treatment effects. This vignette demonstrates:

  1. How partial pooling controls false positives by pulling weak/uncertain effects toward zero
  2. How to prioritize compounds for follow-up using probability of effect (\(\pi\)) and effect size (\(\rho'\))

2 Simulating data with known ground truth

2.1 Why simulate?

To demonstrate how partial pooling works, we need data where we know the true effect sizes in advance. This allows us to compare what the model estimates against what we know is actually true. In real experiments, we never know the ground truth—simulation gives us that visibility.

2.2 Simulation design

We generate synthetic cell migration data that mimics a realistic high-throughput screen:

  • 306 treatment groups: 300 with no/small effect (null), 6 with strong negative effects, 6 with strong positive effects
  • 5 biological replicates (plates) per treatment
  • 3 technical replicates (wells) per plate
  • 25 cells per well

We simulate this design under three noise conditions to show how partial pooling adapts to data quality:

Condition Biological Noise (\(\sigma_{\text{bio}}\)) Technical Noise (\(\sigma_{\text{tech}}\)) Interpretation
Low noise 0.1 0.1 Well-controlled high-throughput screen
High noise 0.3 0.3 Slightly noisier high-throughput screen
Very high noise 1.0 1.0 Challenging conditions (e.g., experiment perform under heterogeneous conditions, by multiple labs or individuals)

2.4 Preparing data for analysis

The gen_partial() function returns a list containing the simulated velocity data ($y) and the true parameters. We extract and format the data frames for analysis:

# Extract and format the velocity data frames
format_sim_data <- function(sim_list) {
    d <- sim_list$y
    d$well <- as.character(d$well)
    d$compound <- as.character(d$compound)
    d$plate <- as.character(d$plate)
    d$offset <- 0
    d$offset[d$group == 1] <- 1  # Mark control group for batch correction
    return(d)
}

yl <- format_sim_data(yl)
yh <- format_sim_data(yh)
yvh <- format_sim_data(yvh)

What was simulated? Each dataset contains cell velocity measurements with known true treatment effects (delta_t). The three datasets differ only in their noise levels, allowing us to isolate how uncertainty affects effect estimation.

4 Partial pooling enables robust inferrence and controls false positives

4.1 What is partial pooling?

The key advantage of hierarchical modeling is partial pooling: treatment effects with high uncertainty are shrunk toward more stable estimates, borrowing strength from related groups in the data hierarchy.

cellmig implements partial pooling at three hierarchical levels, matching the structure of high-throughput migration assays:

Level What is pooled From → To Why it matters
1. Cell → Well Individual cell velocities Cells → Well mean (\(\mu_w\)) Stabilizes well estimates when few cells are tracked
2. Well → Plate Well-specific treatment effects Wells → Plate-specific effect (\(\gamma_{pt}\)) Accounts for technical variability between wells
3. Plate → Treatment Plate-specific treatment effects Plates → Overall treatment effect (\(\delta_t\)) Accounts for biological variability between replicates
4. Treatment → Population Overall treatment effects All treatments → population of treatments: N(0, \(\sigma_\delta\)) Controls false positives via shrinkage toward null

When data is noisy or sparse, the population-level information dominates (strong shrinkage). When data is abundant and consistent, the group-specific data dominates (weak shrinkage).

In this vignette: We focus on the top level (treatment → population), where overall treatment effects (\(\delta_t\)) are shrunk toward zero. This is the mechanism that automatically controls false positives in large screens without requiring explicit multiple testing correction.

4.3 Interpreting the shrinkage plot

Pattern What It Means Why It Matters
Points near diagonal Effects estimated similarly across noise levels Strong, well-supported effects—reliable regardless of noise
Points pulled toward zero on x-axis Weak effects in high-noise data shrunk toward null Partial pooling prevents over-interpreting noisy data
Wider error bars on x-axis Higher uncertainty in high-noise conditions Model correctly reports less confidence when data is noisy

Key insight: Points that deviate from the diagonal represent treatments where noise level affects the estimate. Notice how points cluster near the diagonal for large effects (strong signals) but scatter toward zero for small effects (weak signals) in the high-noise panels. This is partial pooling in action.

This shrinkage mechanism automatically controls false positives without requiring explicit multiple testing correction. Treatments with weak evidence are conservatively estimated near zero, while strong effects remain detectable.

5 Prioritizing compounds for follow-up

In large screens, you need to prioritize which compounds to validate experimentally. We recommend using a volcano-like plot combining effect size (\(\rho'\)) and probability of effect (\(\pi\)).

5.1 Understanding the metrics

Before prioritizing, it’s important to understand what these metrics represent:

5.1.1 Effect Size (\(\rho'\)): magnitude of change

The parameter \(\rho'\) represents the fold-change in migration velocity effects (\(\delta_t\)) between two treatment groups (exponentiated from log-scale):

  • \(\rho' = 1\): No effect (null hypothesis; equal velocity between groups)
  • \(\rho' < 1\): Reduced migration (e.g., \(\rho' = 0.7\) indicates a 30% reduction in velocity)
  • \(\rho' > 1\): Increased migration (e.g., \(\rho' = 1.5\) indicates a 50% increase in velocity)

Interpreting magnitude: The biological relevance of effect size depends on your system and research question. As a general guide:

Effect Size (\(\rho'\)) Interpretation Considerations
0.9 – 1.1 Very small change May be statistically detectable but
biologically trivial
0.7 – 0.9 or 1.1 – 1.3 Small to moderate Often biologically meaningful;
context-dependent
0.5 – 0.7 or 1.3 – 2.0 Moderate to large Clear phenotypic impact;
typically worth follow-up
< 0.5 or > 2.0 Large effect Strong phenotypic change; high priority
if confident

Important: These ranges are illustrative, not prescriptive. In some biological contexts, a 10% change may be highly meaningful (e.g., subtle signaling perturbations), while in others, only dramatic changes warrant attention.

5.1.2 Probability of effect (\(\pi\)): confidence in the estimate

The parameter \(\pi\) quantifies the probability that a true difference exists between treatments, based on the posterior distribution:

  • \(\pi \approx 0\): No evidence for a difference (posterior distribution of \(\rho'\) centered on \(\rho'=1\) with equal density on either side)
  • \(\pi \approx 1\): Strong evidence for a difference (posterior distribution of \(\rho'\) clearly on left or right side of \(\rho'=1\))

Unlike p-values, \(\pi\) is a direct probability statement about the effect, making it more intuitive for decision-making. However, it should always be considered alongside effect size magnitude.

5.2 Creating a prioritization plot

Visualizing both metrics together helps identify compounds that combine meaningful effect sizes with high confidence:

# Get pairwise comparisons against control
pairs <- get_pairs(x = ch, exponentiate = TRUE)$ds

                      xmax = exp(rho_H95)), col = "gray")+

               fill = "white", col = "black")+
    geom_vline(xintercept = 1, linetype = "dashed", col = "red")+
    ylab(label = expression(pi))+
    xlab(label = expression(rho*"'"))+
    scale_x_log10()+
    annotation_logticks(base = 10, sides = "b")

The plot shows effect size (\(\rho'\), x-axis) versus confidence (\(\pi\), y-axis). The vertical dashed line at \(\rho'=1\) indicates the null effect (no change).

5.3 A flexible prioritization framework

Rather than applying rigid thresholds, we recommend considering both metrics jointly when prioritizing compounds. The following framework can guide decision-making.

5.3.1 Key principles for decision-making

  1. High \(\pi\) + \(\rho'\) >> 1 or \(\rho'\) << 1: Best candidates—confident and biologically meaningful
  2. High \(\pi\) + Small \(\rho'\): Statistically real but assess biological relevance before investing resources
  3. Low \(\pi\): Lower priority

5.3.2 Adjusting for your context

The appropriate prioritization strategy depends on several factors:

  • Screen size: Larger screens may require higher \(\pi\) to manage validation workload
  • Validation cost: Expensive follow-up assays warrant higher confidence (\(\pi > 0.99\))
  • Biological context: Some systems expect subtle effects; others require dramatic changes
  • Discovery stage: Early exploration may accept lower \(\pi\) for leads; late-stage validation requires high confidence
  • Effect direction: You may prioritize suppressors over enhancers (or vice versa) based on your biological question

5.5 Validate by visualizing raw data

Before committing resources to experimental validation, we recommend inspecting the raw velocity distributions for top candidates. This helps verify that:

  1. Effects are consistent across biological replicates (plates)
  2. No plate-specific artifacts are driving the signal
  3. Velocity distributions appear reasonable (not skewed by outliers or tracking errors)
# Extract processed data from fitted model
d <- ch$data$proc_x

# Select the top-ranked comparison for visualization
top_hit <- high_confidence[1, ]

# Filter data for the two groups being compared
d <- d[d$group_id %in% c(top_hit$group_id_x, top_hit$group_id_y), ]

# Plot raw velocities by treatment group, faceted by plate
ggplot(data = d)+
    facet_wrap(~plate_id, nrow = 1)+
    geom_sina(aes(x = group, y = v, group = well), size = 0.5, alpha = 0.6)+
    ylab("Cell velocity (µm/min)")+
    xlab("Treatment group")+
    theme_bw(base_size = 10)+
    theme(axis.text.x = element_text(angle = 45, hjust = 1))
